欢迎来到这个新的个人博客。

它运行在 GitHub Pages 和 Jekyll 之上，内容以 Markdown 文件保存。你只需要在 `_posts` 目录新增一篇文章，提交并推送到 GitHub，站点就会自动更新。

## 为什么选择这种方式

个人博客最重要的不是复杂功能，而是能持续写下去。这个站点优先保证三件事：

- 写作简单：文章就是 Markdown 文件，可以用 Cursor、VS Code、Obsidian 或任何文本编辑器书写。
- 浏览舒适：页面是静态 HTML，加载快，适合手机和桌面阅读。
- 风格独特：暗色星空、玻璃卡片和渐变光晕让站点有记忆点，但不会抢走内容的注意力。

## 一篇文章的基本格式

每篇文章都放在 `_posts` 目录，文件名格式为：

```text
YYYY-MM-DD-your-post-title.md
```

文章开头需要一段 front matter：

```yaml
---
title: "文章标题"
description: "一句话介绍这篇文章。"
date: 2026-04-30 10:30:00 +0800
tags: [技术, 项目]
---
```

下面就可以正常写 Markdown。你可以使用二级标题、列表、引用、代码块、表格和图片。

## 示例代码块

```js
const post = {
  title: "新的想法",
  status: "draft",
  publish() {
    return "push to GitHub Pages";
  }
};
```

## 下一步

你可以把这篇示例文章改成真正的第一篇博客，也可以保留它作为写作参考。更推荐的方式是复制 `_drafts/template.md`，改好标题、日期、标签和正文，再移动到 `_posts` 目录发布。
